# バグ報告: ショッピングカートに前の客の商品が混ざる

## 症状
新しいユーザーがカートに最初の商品を入れた瞬間、なぜか前のユーザーがカートに入れていた商品も表示される。
朝イチで使うときは問題なく、しばらく使っているうちに発生する。
サーバーを再起動すると治る。

## 再現
1. ユーザー A でカートに「リンゴ」を追加 → カートは `[リンゴ]` ✅
2. ログアウト
3. ユーザー B でログイン
4. カートに「バナナ」を追加 → カートは `[リンゴ, バナナ]` ❌（リンゴが残っている）

## 関連ファイルの場所
- `src/cart.py` — カート機能

## ほしいもの
原因と修正方針。
