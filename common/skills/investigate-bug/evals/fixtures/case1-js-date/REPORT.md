# バグ報告: 予約日が1日ズレる

## 症状
営業からの報告: 「お客さんが `2024-01-15` で予約したのに、管理画面では `2024-01-14` と表示される」とのこと。
全件ではなく、夜遅くに登録した予約に多い気がする。

## 再現
- 環境: 本番（サーバーは UTC、フロントは Asia/Tokyo）
- フォーム: HTML の `<input type="datetime-local">` を使っており、送信される値は `"2024-01-15T00:00:00"`（タイムゾーン情報なしのローカル時刻）
- API は受け取った文字列を `input.date` として `createBooking()` に渡す
- 結果: 管理画面の `displayDate` が `2024-01-14` になる

## 関連ファイルの場所
- `src/booking.js` — 予約処理のメイン
- `src/format.js` — 日付の整形

## ほしいもの
原因の特定と修正方針。データの直し方も知りたい。
