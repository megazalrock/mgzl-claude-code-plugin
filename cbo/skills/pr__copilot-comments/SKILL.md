# pr:copilot-comments

指定したPRからGitHub Copilotによる未解決のレビューコメントを取得して表示するスキル。

## コンテキスト

- PR番号: $ARGUMENTS

## タスク

1. PR番号が引数で渡されていない場合はユーザーにその旨を通知し終了
2. 以下のスクリプトを実行してCopilotの未解決コメントを取得

```bash
bun run "${CLAUDE_SKILL_DIR}/scripts/fetch-copilot-comments.ts" $ARGUMENTS
```

3. スクリプトの出力はJSON形式。結果に応じて以下のように対応する:
   - `error` フィールドがある場合: エラー内容をユーザーに通知
   - `unresolvedCount` が 0 の場合: 未解決のCopilot指摘がない旨を通知
   - 指摘がある場合: 以下の形式でユーザーに提示

4. **修正要否の検討**（指摘が1件以上ある場合のみ）: 各コメントについて以下の手順で判定する
   1. コメントの `path` と `line` を元に、該当ファイルの該当箇所を Read で読み込む
   2. 以下の観点で **「修正する」** か **「対応不要」** の2段階に分類する。判断が曖昧な場合は周辺コードやプロジェクトの規約を追加調査し、必ずどちらかに確定させる:
      - コード上の実際のバグや問題を指摘しているか
      - 既に修正済み、または現在のコードでは該当しない指摘ではないか
      - プロジェクトの規約・方針と矛盾する指摘ではないか
      - スタイル上の好みレベルの指摘ではないか
   3. 判定結果を「検討結果」フォーマットでユーザーに提示して終了（修正は行わない）

## 出力フォーマット

### コメント一覧

```
### PR #<number>: <title>
<url>

未解決のCopilot指摘: <unresolvedCount>件

---

#### 1. `<path>`:<line info>
<コメント本文（suggestion blockがあればそのまま表示）>

---
(以降繰り返し)
```

- 行情報は `startLine` がある場合は `L<startLine>-L<line>`、ない場合は `L<line>` と表示
- `line` が null の場合はファイルレベルのコメントとして行番号表示を省略

### 検討結果

```
## 検討結果

| # | ファイル | 判定 | 理由 |
|---|---------|------|------|
| 1 | `<path>`:L<line> | 修正する | <判定理由> |
| 2 | `<path>`:L<line> | 対応不要 | <判定理由> |
```
