# test-runner

テストパスに指定したテストファイルまたはディレクトリのテストを実行するスキルです。

## コンテキスト

- テストパス: $ARGUMENTS

## タスク

出力する場合はリストとして装飾を省いて出力します。

0. テストパスが指定されていない場合はユーザーにその旨を通知し終了します
1. `run-test.ts` を実行しテスト結果を取得
2. 全てのテストが成功した場合、テストファイル、テストケース、実行時間を出力して終了
3. いずれかのテストが失敗した場合、失敗したテストのみを全て表示して終了

## `run-test.ts` の実行方法

```bash
bun run "${CLAUDE_SKILL_DIR}/scripts/run-test.ts" <テストパス>
```

- `<テストパス>` (必須): テスト対象のファイルまたはディレクトリのパス
- テストパスを指定せずに実行するとエラーになります
- 全テストの実行は非常に時間がかかるため、必ず対象を絞って実行してください

## デフォルトのテストコマンド

`run-test.ts` は既定で `vitest run --reporter dot` を実行します。

- `dot` reporter は成功テストを1文字に圧縮するため、Claude Code が出力を読む際のトークン消費を最小化できます
- 失敗時のエラー・差分・スタックトレースは reporter に関わらず vitest が完全に出力します
- テストコマンドを変更したい場合は環境変数 `TEST_COMMAND` で上書きできます
