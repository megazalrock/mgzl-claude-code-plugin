## セレクタ記載ガイド

セレクタ情報は `browser_run_code` で直接操作できる形式で記載する。これにより `browser_snapshot` なしで操作を実行できる。

優先順位:
1. ARIAロール + テキスト: `getByRole('button', { name: '確定する' })`
2. ラベルテキスト: `getByLabel('タイトル')`
3. テキスト内容: `getByText('作成しました')`
4. data属性: `locator('[data-testid="item-title"]')`

## Playwrightスクリプト

操作手順（Step）から導出された、直接実行可能なスクリプトをPhase単位で記載する。
スクリプトがある場合、エージェントは解釈・コード生成なしに `browser_run_code` へそのまま渡して実行できる。

> **Stepが正（source of truth）、スクリプトは従（derived）**
> - Stepの操作説明が壊れてもスクリプトから復元でき、スクリプトが壊れてもStepから再構築できる
> - 操作手順を更新したらスクリプトも更新すること

### Phase形式

```markdown
### Phase N: [フェーズ名]
- 対応Step: [対応するStepの名前]
- 区切り理由: [モーダルのDOM生成待ち / API通信 / ドロップダウン操作等]

\```javascript
async ({ page }) => {
  // スクリプト本体
}
\```
```

### プレースホルダー規約

動的な値（ユーザー入力やID等）はプレースホルダーで記載し、実行時にエージェントが置換する。

| 形式 | 説明 | 例 |
|------|------|----|
| `{{UPPER_SNAKE_CASE}}` | 実行時に置換する動的な値 | `{{TITLE}}`, `{{START_DATE}}`, `{{UNIQUE_ID}}` |

- プレースホルダー名はすべて大文字のスネークケースで記載する
- 手順書の冒頭または各Phaseの前に、使用するプレースホルダーの一覧と説明を記載する
