## コンテキスト

このスキルは Claude Code の **Plan Mode 中** に呼ばれることを前提とする。
ユーザーがプランファイルを作成中／作成完了直後に「このプランをレビューして」と指示するためのスキルで、Plan Mode を抜ける前に Markdown 仕様としての品質チェックを行う。

- Plan Mode 中はプランファイル本体の Edit が許可されているため、レビュー結果を反映する自動修正もそのまま実行できる
- レビュー結果はファイルに保存せず、会話に出力するのみ

## タスク

**IMPORTANT**: このスキルは引数を取らない。レビュー対象のプランファイルパスは**会話コンテキストから自動取得**する。

1. **プランファイルパスの取得**
   - 会話コンテキスト（システムリマインダー）の `## Plan File Info:` ブロックを探し、絶対パスを抽出する
   - 抽出パターン: `(?:your plan at|existing plan file at)\s+(/[^\s]+\.md)` にマッチする最初のパス
   - 抽出は Claude 自身が会話履歴を読み返して行う（Bash の `grep` ではコンテキストを検索できない）
   - **抽出失敗時の動作**:
     - 「プランモードで作成されたプランファイルが検出できませんでした。Plan Mode が有効な状態で再度実行してください。」と出力して**即終了**する
     - ユーザーへの聞き返し、パスの推測、`ls ~/.claude/plans/` 等の探索は**禁止**
   - 抽出成功時は、そのパスを以降のステップで `<plan_file_path>` として参照する

2. **プランファイルの読み込み**
   - `Read` ツールで `<plan_file_path>` の内容を全行読み込む
   - 読み込みに失敗した場合は、その旨をユーザーに伝えて即終了する

3. **並列レビュー実行**
   - 以下 2 つのサブエージェントを **同一アシスタントメッセージ内に並列起動** する（Agent ツールを 2 つ並べる）
     - `@reviewer-for-logic`（subagent_type: `reviewer-for-logic`）
     - `@reviewer-for-design`（subagent_type: `reviewer-for-design`）
   - 各エージェントへの prompt には以下を必ず含める
     1. プランファイルの絶対パス `<plan_file_path>`
     2. 「これは Claude Code の Plan Mode で作成された実装プランです。実装着手前の Markdown 仕様レビューを行ってください」という文脈説明
     3. Severity `[5]` 必須修正 / `[4]` 強く推奨 / `[3]` 推奨 / `[2]` 任意 / `[1]` 情報 で分類して報告すること

4. **指摘の集約と判定**
   - 2 つのサブエージェントから返ってきた指摘を統合し、Severity 別に件数を集計する
   - **判定ルール**:
     - `[3]` 以上（`[3]`/`[4]`/`[5]`）の合計が **0 件** → 6. に進む
     - `[3]` 以上が **1 件以上** → 5. に進む
   - `[2]` 以下の指摘は最終サマリには含めるが、自動修正の対象にはしない

5. **自動修正ループ（最大 2 回）**
   - 修正回数カウンタを 0 で初期化し、以下のループを実行する
     1. **修正実施**
        - `[3]` 推奨以上の全指摘について、メインエージェント（このスキル実行中の Claude）が `Edit` ツールで `<plan_file_path>` を直接修正する
        - 修正は**指摘箇所をピンポイントで**行う。プラン全体の書き直しは禁止（プラン作成者の意図を尊重）
        - **プランファイル以外の編集は禁止**（Plan Mode 制約）
        - 修正回数を 1 増やす
     2. **再レビュー**
        - 3. と同じ要領で `@reviewer-for-logic` と `@reviewer-for-design` を並列起動して再レビュー
     3. **判定**
        - `[3]` 以上が **0 件** → ループを抜けて 6. に進む
        - `[3]` 以上が **1 件以上** かつ **修正回数 < 2** → 1. に戻る
        - `[3]` 以上が **1 件以上** かつ **修正回数 ≥ 2** → 4. に進む
     4. **上限到達時のユーザー判断**
        - `AskUserQuestion` で以下 3 択を提示する
          - 「現状のまま完了する」→ 残存指摘を明示して 6. に進む
          - 「もう一度だけ修正を試みる」→ 1. に戻る（再度 4. に到達したら再びユーザー確認）
          - 「中止する」→ 残存指摘のみ提示して終了する

6. **最終レビューサマリの出力**
   - 会話に**箇条書きで最終サマリを表示**する。**ファイル保存はしない**
   - 出力の先頭に、ユーザーが手動で `$MGZL_DIR/reviews/` に保存できるよう、YAML フロントマターを含めて出力する:
     ```yaml
     ---
     reporter: ClaudeCode review:plan
     model: claude-sonnet-4-6   # 自身のモデル名を記載
     ---
     ```
     - `reporter` は固定で `ClaudeCode review:plan`。`model` は実行中の自身のモデル名（不明なら `unknown`）。
     - 各指摘ブロックには `**評価**:` / `**評価理由**:` の空欄も含める（フォーマットは `cbo/skills/document-saver/references/format-review-result.md` を参照）。
   - 出力に含める項目:
     - レビュー対象パス: `<plan_file_path>`
     - レビュー実施回数 / 修正回数
     - 残存指摘（Severity 別。`[3]` 以上 0 件なら「指摘なし」と明示）
     - 参考指摘（`[2]` / `[1]`。任意で折りたたみ表示）
     - 次のアクション提案（例:「このまま実装に進めます」「残存 [3] 指摘の手動対応を推奨」）

## 制約

- このスキルが編集できるのは、コンテキストから抽出した**プランファイル 1 つだけ**
- サブエージェント（`@reviewer-for-logic`, `@reviewer-for-design`）は read-only なので Plan Mode 制約と衝突しない
- レビュー結果は会話出力のみ。`document-saver` 等によるファイル保存は行わない
- 出力言語は日本語
